镜像命令

在仓库搜索镜像

  • -f:根据指定的过滤条件进行搜索
  • --format:指定输出的格式
  • --limit:限制搜索结果的数量
  • --no-trunc:显示完整的镜像描述
  • --stars:根据指定的星级评分筛选搜索结果

下载指定的镜像

  • -a:拉取指定镜像名称的所有标签
  • -q:只显示拉取的镜像 ID
  • --disable-content-trust:禁用内容信任验证

将本地的镜像推送到镜像仓库

  • -q:只显示推送的镜像 ID
  • --tls:与仓库建立 TLS 连接
  • --all-tags:同时推送镜像的所有标签
  • --disable-content-trust:禁用内容信任验证

列出本地上的所有镜像

  • -a:显示所有镜像,包括中间层镜像
  • -q:只显示镜像的 ID
  • --no-trunc:显示完整的镜像 ID
  • --digests:显示镜像的摘要信息
  • --filter:根据指定的条件过滤镜像
  • --format:指定输出的格式
  • REPOSITORY:指定仓库名称
  • TAG:指定镜像的标签

删除指定的本地镜像

  • -f:强制删除镜像
  • -q:只显示被删除的镜像 ID
  • --no-prune:不自动删除与镜像关联的未被使用的中间层镜像

构建一个新的镜像

  • -t:为镜像指定标签
  • -f:指定要使用的 Dockerfile 路径
  • -q:只显示构建过程中生成的镜像 ID
  • --build-arg <variable=value>:为构建过程中的 ARG 指令传递变量和值

为现有的本地镜像添加一个新的标签

显示指定镜像的历史记录

  • --format:指定输出的格式
  • --human:以人类可读的格式显示文件大小
  • --no-trunc:显示完整的命令和结果

将指定的镜像保存到一个tar文件中

从一个tar文件中加载一个镜像

容器命令

创建并运行一个新容器

  • -d:在后台以守护进程方式运行容器
  • -p:将容器的端口映射到主机的端口
  • -v:挂载主机目录或数据卷到容器
  • -e:设置环境变量
  • -it:以交互模式运行容器,并分配一个伪终端
  • --rm:当容器退出时自动删除容器
  • --name:为容器指定一个名称
  • --network:指定容器连接的网络模式

启动一个已创建的容器

  • -a:将标准输入(STDIN)、标准输出(STDOUT)和标准错误输出(STDERR)与容器进行关联
  • -it:以交互模式运行容器,并分配一个伪终端

停止一个正在运行的容器

  • -t:指定停止容器前等待的时间(默认 10s)

重启一个容器

  • -t:指定停止容器前等待的时间(默认 10s)
  • -s:指定用于关闭容器的信号
  • --time-format="<time_format>":指定等待时间的格式

删除一个停止的容器

  • -f:强制删除运行中的容器
  • -v:删除关联的卷
  • -l:删除容器与其他容器的链接

列出正在运行的容器

  • -a:显示所有容器,包括已停止的容器
  • -q:只显示容器的 ID

查看容器的日志

  • -f:实时跟踪容器日志的输出
  • -t:在日志中显示时间戳
  • --tail:仅显示最后几行的日志
  • --details:显示更详细的日志输出

在正在运行的容器中执行命令

  • -i:以交互模式运行容器,并分配一个伪终端
  • -t:分配一个伪终端

检查容器的详细信息

  • -f:指定输出的格式
  • --type:指定要检查的对象类型
  • --size:显示相关对象的大小信息
